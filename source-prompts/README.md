# 源码研究提示词框架 v3.0

面向**漏洞挖掘**、**编码能力提升**和**自驱源码研究**的 AI 源码分析框架。支持 **C/C++、Java、Go、Python** 四种语言，专注于安全研究和高质量代码学习。

> **v3.0 核心升级**：引入总控协议（`00_master.md`）+ MEU 证据标准 + 多轮状态机 + 8 章节统一输出 Schema，使 AI 研究深度更高、输出更稳定、多轮可累积。

## v3.0 核心改进

### 1. 总控协议（`00_master.md`）

新增统一总控层，将 01-32 从"模块池"升级为"研究协议"：

- **阶段调度**：强制按 6 阶段顺序执行，消除跳步
- **状态管理**：每轮输出 `confirmed/hypothesis/refuted/unknown` 状态
- **失败判定**：MEU < 3/5 强制 hypothesis，禁止 confirmed
- **收敛检查**：自动判断研究是否达到边界

### 2. MEU 证据标准

引入**最小证据单元（MEU）**模型，强制每结论必须有 5 类证据中的至少 3 类：

| 证据类型 | 说明 |
|---------|------|
| 文件路径 | 精确到文件名（最好到行号） |
| 函数/方法名 | 完整的函数签名 |
| 分支条件 | if/switch/条件判断的具体条件 |
| 数据结构字段 | 结构体字段名及其类型/用途 |
| 调用关系 | 函数 A → 函数 B 的调用关系 |

**防泛化机制**：不满足 MEU → 强制 hypothesis → 禁止确定性结论

### 3. 多轮状态机

研究结论可持续累积，避免每轮重开：

```
Round N 结束 → MEU 判定 → 状态更新 → 生成 Round N+1 问题池
→ 若未收敛 → 返回阶段 1 → 继续
→ 若收敛 → 输出最终报告
```

### 4. 统一 8 章节输出 Schema

所有模块输出共享同一结构，支持自动拼接与多轮累积：

1. 研究问题清单
2. 调用链（主路径 + 异常/失败路径）
3. 关键函数卡片
4. 关键数据结构卡片
5. 设计决策与权衡
6. 漏洞假设与验证
7. 未解问题（下一轮优先级）
8. 多轮状态更新

## 使用方法

### 方法一：通过总控协议自动调度（v3.0 推荐）

适用于深度源码研究和漏洞挖掘，AI 自动按阶段推进：

```
阶段 0（初始化）→ 阶段 1（问题生成）→ 阶段 2（调用链追踪）
→ 阶段 3（安全分析）→ 阶段 4（漏洞验证）→ 阶段 5（研究综合）
→ 阶段 6（收敛判定）
→ 若未收敛 → 返回阶段 1（读取上一轮状态）→ 循环
```

**优势**：
- 自动管理研究状态，不重复分析
- MEU 判定保证证据质量
- 多轮累积，研究深度可持续推进

### 方法二：直接调用模块（保留，向后兼容）

适用于快速查询或单点分析：

```
阶段一（01-06）：快速理解项目
  01 → 02 → 03 → 04 → 05 → 06

阶段二（07-16）：深入安全分析
  07 → 08 → 09 → 10 → 11 → 12 → 13 → 14 → 15 → 16

阶段三（17-21）：架构与代码流
  17 → 18 → 19 → 20 → 21

阶段四（22-27）：编码技巧 + 报告
  22 → 23 → 24 → 25 → 26 → 27
```

### 方法三：快速安全审计（仅漏洞挖掘）

```
07（攻击面）→ 08（输入验证）→ 09（协议安全）→ 10（内存安全）
→ 11（加密认证）→ 12（并发）→ 13（注入）→ 14（配置）
→ 15（利用路径）→ 16（安全报告）
```

### 方法四：快速学习编码（仅技巧提炼）

```
22（内存管理）→ 23（数据结构）→ 24（多态抽象）
→ 25（性能优化）→ 26（跨平台）→ 27（编码规范）
```

## 注意事项

1. **总控优先**：`00_master.md` 的指令优先级最高，任何模块的局部指令与总控冲突时，以总控为准
2. **材料决定质量**：提供的源码材料越完整，AI 分析越准确
3. **语言自适应**：每个提示词开头会判断语言类型（C/Java/Go/Python），后续分析自动适应
4. **安全优先**：漏洞挖掘时请务必完成 02_SECURITY 全部 10 个提示词
5. **实战结合**：使用 07_PRACTICAL 进行实际操作验证
6. **技巧复用**：06_TECHNIQUES 的输出可直接作为代码编写参考
7. **分步深入**：不要一次性提供太多材料，分步使用效果更好
8. **MEU 不可绕过**：在任何情况下，禁止在无 3/5 类 MEU 的情况下输出 `confirmed`
9. **多轮累积**：使用 `00_master.md` 时，每轮结束后必须更新状态文件，作为下一轮的输入
10. **研究有边界**：通过收敛条件（confirmed / unknown ≤ 2 / budget 用尽）控制研究深度

## 设计原则

- **多语言支持**：C/C++、Java、Go、Python 自动识别，漏洞模式和分析重点按语言切换
- **安全第一**：漏洞挖掘是核心目标，每个安全提示词都包含 PoC
- **证据驱动**：每个发现都要给出具体的代码位置和证据，MEU 强制执行
- **问题驱动**：先生成高价值代码级问题，再逐题追踪解答，31→32 形成闭环
- **语言无关**：扩展点分析（28-30）为通用方法论，不绑定特定框架或语言
- **状态累积**：多轮研究结论可持续累积，避免每轮重开
- **研究有边界**：通过收敛条件控制研究深度，避免无限深入
