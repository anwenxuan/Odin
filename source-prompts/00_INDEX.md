## v3.0 核心概念速查

### MEU（最小证据单元）

每个结论必须包含以下 5 类证据中的**至少 3 类**：

| 证据类型 | 说明 |
|---------|------|
| 文件路径 | 精确到文件名（最好到行号） |
| 函数/方法名 | 完整的函数签名 |
| 分支条件 | if/switch/条件判断的具体条件 |
| 数据结构字段 | 结构体字段名及其类型/用途 |
| 调用关系 | 函数 A → 函数 B 的调用关系 |

**判定规则**：
- MEU < 3/5 → 强制 `hypothesis`，禁止 `confirmed`
- MEU ≥ 3/5 且无反例 → `confirmed`
- 存在反例 → `refuted`

### 多轮状态

| 状态 | 定义 |
|------|------|
| `confirmed` | 证据充分且无反例，进入最终报告 |
| `hypothesis` | 证据不完整，进入下一轮优先队列 |
| `refuted` | 发现反例，归档为"已排除" |
| `unknown` | 当前阶段无法回答，进入问题池待补充材料 |

### 统一输出 Schema（8 章节）

1. 研究问题清单
2. 调用链（主路径 + 异常/失败路径）
3. 关键函数卡片
4. 关键数据结构卡片
5. 设计决策与权衡
6. 漏洞假设与验证
7. 未解问题（下一轮优先级）
8. 多轮状态更新

### 收敛条件

满足以下任一条件，研究收敛：
- 所有 P0 高优先级问题 `confirmed`
- 剩余 `unknown` ≤ 2 且需要特殊材料（二进制/运行态）
- research budget 用尽

## 占位符说明

| 占位符 | 说明 | 获取方式 |
|--------|------|---------|
| `{rubric}` | README 内容 | 读取 README.md |
