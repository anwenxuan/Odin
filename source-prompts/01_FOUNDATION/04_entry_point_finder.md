## 分析任务

### 1. 入口文件识别

找到以下类型的入口文件：

| 文件类型 | 识别到的文件 | 说明 | 安全相关 |
|----------|-------------|------|---------|
| 主入口 | main.c / main.py / main.go | 程序启动点 | 高（启动时的安全检查） |
| 应用入口 | app.py / server.js | Web/桌面应用入口 | 高 |
| CLI 入口 | cli.py / bin/ 命令脚本 | 命令行工具入口 | 中 |
| 安全入口 | security.c / auth.c | 安全模块初始化 | 极高 |
| 测试入口 | conftest.py / setup.js | 测试框架入口 | 低 |
| 配置入口 | config.c / settings.py | 配置加载 | 高（影响所有后续安全决策） |

### 4. 启动参数与配置

| 参数/配置 | 来源 | 说明 | 默认值 | 安全影响 |
|----------|------|------|--------|---------|
| --port | 命令行 | 服务端口 | 8080 | 中（暴露面） |
| --bind | 命令行 | 绑定地址 | 0.0.0.0 | 高（是否允许外部访问） |
| --auth | 命令行 | 是否启用认证 | no | 极高 |
| --tls | 命令行 | 是否启用 TLS | no | 极高 |
| --user | 命令行 | 运行用户 | root | 极高（权限过大） |

### 5. 安全初始化序列分析

**安全关键的初始化顺序**：

1. **配置加载**：是否在启动时就限制了不安全选项？
2. **日志初始化**：是否记录了启动过程（安全审计）？
3. **加密模块初始化**：使用的算法是否安全？密钥从哪里来？
4. **认证模块初始化**：认证策略是什么？是否有默认口令？
5. **权限模块初始化**：默认权限是否最小化？
6. **网络监听**：是否监听在安全的地址？是否有暴露的风险？
7. **启动完成**：服务启动后是否做了安全检查？

### 6. 生命周期钩子

识别程序生命周期中的关键钩子点：
- 启动前（before_start / on_startup）
- 启动后（after_start / on_ready）
- 关闭时（on_shutdown / cleanup）
- **安全事件钩子**：认证失败、权限违规、可疑活动
