## 分析任务

### 1. 生成结构树（ASCII）

将目录树重新组织为清晰的层次结构，使用 ASCII 字符：

```
project/
├─ src/
│  ├─ core/       ← 核心业务逻辑
│  ├─ modules/    ← 功能模块
│  └─ utils/      ← 工具函数
├─ tests/          ← 测试代码
├─ configs/        ← 配置文件
└─ docs/           ← 文档
```

注意：用文字说明每个目录的主要职责，标注其重要性（★~★★★★★）。

### 2. 安全敏感目录识别

| 目录 | 职责 | 重要性 | 安全敏感度 | 说明 |
|------|------|--------|-----------|------|
| src/ | 源代码根目录 | ★★★★★ | 高 | 核心代码 |
| security/ | 安全相关 | ★★★★★ | 极高 | 认证/加密/鉴权 |
| crypto/ | 加密实现 | ★★★★ | 极高 | 敏感数据处理 |
| ... | ... | ... | ... | ... |

**安全敏感度标注**：
- **极高**：安全关键代码（认证、加密、鉴权、输入校验）
- **高**：协议解析、网络 I/O、文件操作
- **中**：业务逻辑、数据处理
- **低**：工具、辅助代码

### 3. 目录职责表

| 目录 | 职责 | 类型 | 重要性 | 备注 |
|------|------|------|--------|------|
| src/ | 源代码根目录 | 核心 | ★★★★★ | |
| tests/ | 测试代码 | 测试 | ★★★★ | |
| configs/ | 配置文件 | 配置 | ★★★★ | |
| ... | ... | ... | ... | |

目录类型分类：
- **核心**（放业务逻辑）
- **测试**（测试代码）
- **配置**（配置文件）
- **构建**（构建/部署脚本）
- **文档**（文档/示例）
- **依赖**（第三方代码）
- **资源**（静态资源）
- **安全**（安全相关代码）

### 4. 文件类型统计

统计各类型文件的数量和占比：

| 类型 | 扩展名 | 数量 | 说明 |
|------|--------|------|------|
| 源代码 | .c / .py / .js / ... | N | |
| 测试代码 | test_*.* / *_test.* | N | |
| 配置文件 | .yaml / .json / .toml | N | |
| 头文件 | .h / .hpp / .hxx | N | C/C++ 项目 |
| 文档 | .md / .rst | N | |

### 5. 命名约定分析

识别项目使用的命名约定：
- 文件命名：小写下划线 / 小写连字符 / PascalCase / 其他
- 目录命名：同上
- 示例文件
- **安全相关命名**：安全模块是否使用一致的命名规范
