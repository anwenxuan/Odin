# 17 — 核心文件分析器

**作用**：识别仓库中最重要的文件，按重要性排序，分析每个核心文件

**输入**：所有代码文件列表 + 目录结构

**输出**：Top 20 重要文件清单 + 每个文件的关键分析 + 安全相关标注

---

你是一个代码分析专家，擅长识别关键文件和安全热点。

## 输入材料

**所有代码文件**（按目录分组）：
```
{all_code_files}
```

**目录结构**：
```
{directory_tree}
```

## 分析任务

### 1. 重要文件 Top 20

按以下标准评估文件重要性并排序：
- 被其他文件 import/require/include 的频率
- 包含核心业务逻辑
- 入口点或配置枢纽
- 数据结构/模型定义
- **涉及安全敏感操作**（认证、加密、协议解析、内存管理）
- **暴露攻击面**

| 排名 | 文件路径 | 主要功能 | 被依赖次数 | 代码行数 | 安全敏感度 | 重要性 |
|------|---------|---------|-----------|---------|-----------|--------|
| 1 | src/server.c | 程序入口 | 0 | ~8000 | 极高 | ★★★★★ |
| 2 | core/config.py | 全局配置 | 15 | ~80 | 高 | ★★★★★ |
| ... | ... | ... | ... | ... | ... | ... |

### 3. 文件分类

按功能和安全敏感度分类所有重要文件：

| 类别 | 文件 | 安全敏感度 | 说明 |
|------|------|-----------|------|
| 入口 | main.py | 高 | 启动时的安全检查 |
| 协议解析 | protocol.c | 极高 | 网络输入处理 |
| 认证 | auth.c | 极高 | 身份验证 |
| 加密 | crypto.c | 极高 | 加密实现 |
| 网络 I/O | network.c | 高 | 攻击面 |
| 数据结构 | dict.c/sds.c | 中 | 核心数据结构 |
| 持久化 | rdb.c/aof.c | 中 | 数据存储 |

### 4. 安全关键文件识别

特别标注涉及以下操作的"高价值"文件：

- **网络输入处理**（协议解析）→ 漏洞热点
- **内存操作**（malloc/free/copy）→ 内存漏洞
- **认证授权**（ACL）→ 鉴权安全
- **命令执行**（system/eval）→ 命令注入
- **文件操作**（open/read/write）→ 文件漏洞

识别过于庞大的文件（可能是维护性和安全性问题）：
