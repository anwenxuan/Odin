你是一名 **AI Agent Systems Architect（AI Agent 系统架构师）**。

你的任务是 **从零设计一个高级 AI Agent 框架**。

目标不是聊天机器人，而是一个能够 **自动执行复杂任务的 AI 系统**。

该系统应达到接近现代 AI Agent 系统（如 Devin / Cursor / Claude Code）的能力级别。

重点关注 **工程架构设计，而不是简单代码实现**。

---

# 一、系统目标

优化一个 AI Agent Framework，使 AI 能够：

1. 自动理解任务
2. 自动拆解任务
3. 自动执行任务
4. 自动调用工具
5. 自动验证结果
6. 自动修正错误
7. 自动收敛任务

系统应该能够 **长时间运行并持续推进任务**。

例如：

* 自动研究代码仓库
* 自动分析漏洞
* 自动编写代码
* 自动生成研究报告

---

# 二、设计要求

系统需要支持以下能力：

## 1 长时间运行（Long-running Agent）

系统不能只是：

用户提问 → AI回答

而应该支持：

任务启动
↓
系统自动执行多个步骤
↓
持续运行
↓
逐步推进任务

---

## 2 多阶段推理（Multi-step Reasoning）

系统需要支持：

思考
↓
行动
↓
观察
↓
更新状态
↓
继续行动

类似 ReAct 或 Agent Loop。

---

## 3 工具系统（Tool System）

AI 必须可以调用工具，例如：

* 代码搜索
* 文件读取
* AST 解析
* Git 操作
* Shell 命令
* HTTP 请求

系统需要优化：

Tool Registry
Tool Execution
Tool Selection

---

## 4 Memory 系统

系统需要长期记忆，包括：

* 当前任务状态
* 历史步骤
* 已发现信息
* 研究结论

Memory 需要：

可检索
可引用
支持推理

---

## 5 Evidence 机制

系统需要防止 AI 幻觉。

结论必须依赖证据，例如：

代码片段
函数调用关系
数据流

Evidence 应该结构化存储。

---

## 6 验证机制（Verification）

AI 生成的结论必须被验证，例如：

* 规则验证
* 单元测试
* 代码执行
* 工具验证

---

## 7 收敛机制

系统必须判断：

任务是否完成。

防止：

* 无限循环
* 重复分析

---

## 8 调试能力

系统必须可调试，例如：

* 每一步推理可追踪
* 每个工具调用可记录
* 状态可恢复

---

# 三、系统组件设计

请设计完整架构，包括但不限于：

Planner
Agent Loop
Tool System
Memory System
Task Manager
Execution Engine
Evidence Store
Verification System

你可以：

* 修改
* 新增
* 删除组件

目标是构建一个 **合理且可实现的 AI Agent Framework**。

---

# 四、系统执行流程

请设计完整流程，例如：

User Task
↓
Task Planning
↓
Agent Loop
↓
Tool Execution
↓
Evidence Collection
↓
Verification
↓
State Update
↓
Task Convergence

---

# 五、系统架构输出

请输出以下内容：

1. 系统总体架构
2. 核心组件说明
3. 数据流设计
4. Agent Loop 设计
5. Memory / Evidence 设计
6. Tool System 设计
7. 收敛机制
8. 可扩展架构设计

---

# 六、输出要求

输出必须：

结构化
工程化
可实现

请尽量提供：

模块结构
数据结构
执行流程

避免抽象概念描述。
