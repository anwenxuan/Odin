CLI Reference

Complete command-line interface reference for Odin v0.2.0.

Global Options

These flags apply to all Odin commands.

Option Description Default
--provider LLM provider name: openai, anthropic, azure, ollama, mock mock
--model Model identifier (e.g., gpt-4o, claude-3-5-sonnet) Provider default
--config Path to odin.yaml config file ./odin.yaml
--output, -o Output directory for results ./odin-output/
--verbose, -v Enable verbose logging (INFO level) false
--debug Enable debug logging (DEBUG level) false
--help, -h Show help for the command —
--version Print Odin version —

odin analyze

Run a vulnerability analysis on a target — local directory, file, or Git URL.

bash 
odin analyze <target> [options]

Parameters

Parameter Type Description
<target> string Path to local directory/file, or GitHub URL (e.g., git@github.com:user/repo.git)
--provider string LLM provider override. Choices: openai, anthropic, azure, ollama, mock
--model string Model identifier (e.g., gpt-4o-mini, claude-3-haiku)
--workflow string Workflow to use. Choices: deep-audit (default), quick-audit, surface-scan
--output, -o string Output directory for results
--timeout integer Maximum analysis time in seconds (default: 300)
--format string Output format: json, html, markdown (default: json)
--include string[] File patterns to include (glob), e.g. "**/*.py"
--exclude string[] File patterns to exclude (glob), e.g. "tests/**"
--cache boolean Enable analysis caching (default: true)
--no-rag flag Disable RAG retrieval (faster but less contextual)

Examples

bash 
# Quick scan with Mock mode (no API key needed)
odin analyze ./src

# Deep audit with OpenAI GPT-4o
odin analyze ./src --provider openai --model gpt-4o --workflow deep-audit

# Analyze a specific subdirectory
odin analyze ./src/auth --output ./odin-output/auth

# Remote Git repository
odin analyze https://github.com/user/private-repo --workflow quick-audit

# Custom file patterns, markdown output
odin analyze ./src \
  --include "**/*.py" \
  --exclude "tests/**" \
  --format markdown \
  -o ./odin-report

odin skills

List available skills or show detailed information about a specific skill.

bash 
odin skills list
odin skills show <name>

Parameters — list

Parameter Type Description
--installed flag Show only installed (local) skills
--community flag Show community-contributed skills from the registry
--language string Filter skills by language (e.g., python, javascript)
--priority string Filter by priority: p0, p1, p2

Parameters — show

Parameter Type Description
<name> string Skill name (e.g., detect-sql-injection)
--format string Output format: json, yaml, markdown (default: yaml)

Examples

bash 
# List all available skills
odin skills list

# Show details of a specific skill
odin skills show detect-sql-injection

# List only Python skills
odin skills list --language python --priority p0

odin workflow

List available workflows or execute a specific workflow on a target.

bash 
odin workflow list
odin workflow run <name> <target> [options]

Available Workflows

Workflow Skills Use Case
deep-audit 12 skills, all priorities Comprehensive vulnerability research. DAG-layer parallel execution.
quick-audit 5 skills, P0+P1 only Fast scan for critical and high severity issues. CI-friendly.
surface-scan 3 skills, P0 only Lightweight surface-level check. Sub-60s runtime.

Parameters — run

Parameter Type Description
<name> string Workflow name: deep-audit, quick-audit, or surface-scan
<target> string Path or Git URL to analyze
--output string Output directory
--parallel boolean Enable parallel skill execution (default: true)

Examples

bash 
# List all available workflows
odin workflow list

# Run a quick-audit on a local directory
odin workflow run quick-audit ./src/auth

# Run a deep-audit with parallel execution disabled
odin workflow run deep-audit ./src --parallel false -o ./odin-output/full

odin eval

Run benchmark evaluations or generate evaluation reports from existing results.

bash 
odin eval run [--dataset <path>] [--output <dir>]
odin eval report [--results <path>] [--format <fmt>]

Parameters — run

Parameter Type Description
--dataset string Path to evaluation dataset (JSON/YAML). Defaults to the built-in benchmark set.
--output string Output directory for evaluation results
--metrics string[] Metrics to compute: precision, recall, f1, meu-pass-rate
--provider string Provider for evaluation runs

Parameters — report

Parameter Type Description
--results string Path to evaluation results JSON
--format string Report format: json, html, markdown (default: html)
--output string Output file path

Examples

bash 
# Run evaluation on the default benchmark dataset
odin eval run --metrics precision recall f1 meu-pass-rate

# Run on a custom dataset
odin eval run --dataset ./benchmarks/my-dataset.json --output ./eval-results

# Generate a markdown report from results
odin eval report --results ./eval-results/results.json --format markdown

odin config

Show the active configuration or set a configuration value.

bash 
odin config show [--key <path>]
odin config set <key> <value>
odin config init [--path <path>]

Parameters — show

Parameter Type Description
--key string Dot-notation path to a specific config value (e.g., provider, rag.chunk_size)
--format string Output format: json, yaml (default: yaml)

Parameters — set

Parameter Type Description
<key> string Configuration key in dot notation (e.g., provider, rag.storage_path)
<value> string New value for the key
--global flag Apply to global config (~/.odin/config.yaml) instead of local

Examples

bash 
# Show the full active configuration
odin config show

# Show a specific config value
odin config show --key provider

# Set the LLM provider
odin config set provider openai

# Set RAG chunk size (globally)
odin config set rag.chunk_size 512 --global

# Initialize a new odin.yaml in the current directory
odin config init
