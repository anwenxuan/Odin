API / REST API

REST API Reference

FastAPI-powered HTTP API for integrating Odin into your pipelines, UIs, and CI systems.

Overview

The Odin REST API is a FastAPI application. Start the server with odin api serve.

bash 
$ odin api serve
INFO:     Uvicorn running on http://localhost:8000
INFO:     Documentation at http://localhost:8000/docs
Header Value Notes
Content-Type application/json Required for all request bodies
Authorization Bearer <token> Optional. Set via --api-key on server start

POST /analyze

Submit a new code analysis job. Returns immediately with a job ID for polling.

POST /analyze

Request Body

json 
{
  "target": "./src",
  "provider": "openai",
  "model": "gpt-4o-mini",
  "workflow": "quick-audit",
  "include": ["**/*.py"],
  "exclude": ["tests/**"],
  "timeout": 300,
  "use_rag": true
}
Field Type Required Description
target string Yes Path or Git URL to analyze
provider string No LLM provider. Default: mock
model string No Model identifier. Default: provider default
workflow string No Workflow name. Default: deep-audit
include string[] No File patterns to include (glob)
exclude string[] No File patterns to exclude (glob)
timeout integer No Max duration in seconds. Default: 300
use_rag boolean No Enable RAG retrieval. Default: true

Response

json 
{
  "analysis_id": "anal_01j8x2k3m4n5p6q7r8s9t0u1",
  "status": "queued",
  "target": "./src",
  "workflow": "quick-audit",
  "created_at": "2026-03-26T10:30:00Z"
}

GET /analysis/{id}

Retrieve the status and results of an analysis job by its ID.

GET /analysis/{id}

Path Parameters

Parameter Type Description
id string The analysis_id returned from POST /analyze

Response — Queued / Running

json 
{
  "analysis_id": "anal_01j8x2k3m4n5p6q7r8s9t0u1",
  "status": "running",
  "progress": 0.45,
  "current_node": "skill:dependency-graph",
  "started_at": "2026-03-26T10:30:05Z",
  "elapsed_seconds": 42
}

Response — Completed

json 
{
  "analysis_id": "anal_01j8x2k3m4n5p6q7r8s9t0u1",
  "status": "completed",
  "completed_at": "2026-03-26T10:35:12Z",
  "elapsed_seconds": 312,
  "summary": {
    "total_findings": 7,
    "critical": 1,
    "high": 2,
    "medium": 3,
    "low": 1,
    "meu_pass_rate": 0.86
  },
  "output_path": "./odin-output/anal_01j8x2k3m4n5p6q7r8s9t0u1/analysis.json"
}
Poll every 5–10 seconds. The full analysis report is written to disk and referenced by output_path.

GET /skills

Return a list of all available skills with their metadata.

GET /skills

Query Parameters

Parameter Type Description
language string Filter by language (e.g., python, javascript)
priority string Filter by priority: p0, p1, p2
installed_only boolean If true, return only locally installed skills

Response

json 
{
  "skills": [
    {
      "name": "detect-sql-injection",
      "description": "Detects SQL injection vulnerabilities in string concatenations",
      "language": "python",
      "priority": "p0",
      "trigger": "**.py",
      "meu_required": true,
      "installed": true,
      "source": "builtin"
    },
    {
      "name": "detect-xss",
      "description": "Detects Cross-Site Scripting in HTML/JS string handling",
      "language": "javascript",
      "priority": "p0",
      "trigger": "**.{js,ts,jsx,tsx}",
      "meu_required": true,
      "installed": true,
      "source": "builtin"
    }
  ],
  "total": 12
}

POST /eval

Trigger a benchmark evaluation run against a dataset.

POST /eval

Request Body

json 
{
  "dataset": "./benchmarks/security-benchmark-v1.json",
  "provider": "openai",
  "model": "gpt-4o",
  "metrics": ["precision", "recall", "f1", "meu-pass-rate"]
}
Field Type Required Description
dataset string No Path to dataset. Defaults to the built-in benchmark.
provider string No Provider to use for evaluation. Default: mock
model string No Model to evaluate
metrics string[] No Metrics to compute. All computed if omitted.

Response

json 
{
  "eval_id": "eval_01j8y3a4b5c6d7e8f9g0h1i",
  "status": "queued",
  "dataset": "security-benchmark-v1",
  "dataset_size": 50,
  "metrics": ["precision", "recall", "f1", "meu-pass-rate"],
  "created_at": "2026-03-26T10:45:00Z"
}

GET /health

Health check endpoint. Returns the server status and component health.

GET /health

Response — Healthy

json 
{
  "status": "healthy",
  "version": "0.2.0",
  "uptime_seconds": 3724,
  "components": {
    "llm": { "status": "ok", "provider": "openai" },
    "mcp": { "status": "ok", "connected": true },
    "rag": { "status": "ok", "vector_count": 1420 },
    "storage": { "status": "ok", "output_dir": "./odin-output" }
  }
}

Response — Degraded

json 
{
  "status": "degraded",
  "version": "0.2.0",
  "uptime_seconds": 184,
  "components": {
    "llm": { "status": "ok", "provider": "openai" },
    "mcp": { "status": "error", "message": "Connection refused" },
    "rag": { "status": "ok", "vector_count": 0 },
    "storage": { "status": "ok" }
  }
}
