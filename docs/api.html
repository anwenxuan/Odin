FastAPI Server

Odin ships with a FastAPI HTTP server for team collaboration. Instead of running CLI commands locally, team members can submit analysis requests via the REST API, making it ideal for integration with CI/CD pipelines, dashboards, and shared research workflows.

ℹ

The API server uses the same analysis engine as the CLI. Set your OPENAI_API_KEY or ANTHROPIC_API_KEY environment variable before starting the server.

Starting the Server

cli/commands/serve.py
# Basic start
python -m uvicorn cli.commands.serve:app --reload --port 8080

# With explicit host/port
python -m uvicorn cli.commands.serve:app --host 0.0.0.0 --port 8080

# Production with multiple workers
python -m uvicorn cli.commands.serve:app --workers 4 --port 8080 --host 0.0.0.0
ℹ

The server runs on port 8080 by default. Use --reload only in development — it enables auto-reload on code changes.

POST /analyze

Submit a new repository analysis. The request is queued and processed asynchronously. Returns a run_id immediately that you can poll with GET /runs/{run_id}.

Request
POST /analyze
Content-Type: application/json

{
  "repo_url": "https://github.com/owner/repo",
  "workflow": "vulnerability_research",
  "provider": "openai",
  "model": "gpt-4o-mini",
  "focus_paths": ["src/auth", "src/api"],
  "max_iterations": 20
}
Response — 202 Accepted
{
  "run_id": "run_abc123def456",
  "status": "queued",
  "message": "Analysis queued. Poll GET /runs/run_abc123def456 for status.",
  "workflow": "vulnerability_research",
  "repo_url": "https://github.com/owner/repo"
}

Request Body Fields

FieldTypeRequiredDescription
repo_urlstringYesGitHub URL or local path
workflowstringYesvulnerability_research, codebase_research, architecture_analysis
providerstringNoLLM provider (default: from env)
modelstringNoModel name (default: gpt-4o-mini)
focus_pathsarray[string]NoPaths to focus analysis on
max_iterationsintegerNoMax iterations per step (default: 20)
branchstringNoGit branch (GitHub repos only)

Python API Client

Use the built-in Python client for programmatic access:

client.py
import requests
import time

BASE_URL = "http://localhost:8080"

def submit_analysis(repo_url, workflow="vulnerability_research"):
    resp = requests.post(f"{BASE_URL}/analyze", json={
        "repo_url": repo_url,
        "workflow": workflow,
        "provider": "openai",
        "model": "gpt-4o-mini",
    })
    resp.raise_for_status()
    return resp.json()

def poll_until_done(run_id, poll_interval=5, timeout=600):
    start = time.time()
    while time.time() - start < timeout:
        resp = requests.get(f"{BASE_URL}/runs/{run_id}")
        resp.raise_for_status()
        data = resp.json()
        status = data["status"]
        if status == "succeeded":
            return data["report"], data["evidence_stats"]
        elif status == "failed":
            raise RuntimeError(f"Analysis failed: {data['error']}")
        print(f"Status: {status} — waiting {poll_interval}s...")
        time.sleep(poll_interval)
    raise TimeoutError(f"Analysis timed out after {timeout}s")

# Usage
result = submit_analysis("https://github.com/owner/repo")
report, evidence = poll_until_done(result["run_id"])
print(f"Report generated with {evidence['total_meus']} evidence units")

CORS & Authentication

CORS Configuration

Enable CORS for browser-based clients:

terminal
python -m uvicorn cli.commands.serve:app \
  --port 8080 \
  --proxy-headers \
  --forwarded-allow-ips='*'

Authentication (Production)

For production deployments, consider adding authentication via a reverse proxy (nginx, Traefik) or middleware:

⚠

The default server configuration has no authentication. Do not expose the API server directly to the internet without adding auth. Use a VPN, reverse proxy with basic auth, or API key middleware.
