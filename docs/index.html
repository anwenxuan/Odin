Key Features

Autonomous Agent

LLM-powered agents that call real tools to read files, search code, run shell commands, and execute Git operations.

DAG Parallel Execution

Independent workflow steps run concurrently via ThreadPoolExecutor, maximizing throughput across CPU cores.

Evidence-Based

Every conclusion is backed by a Minimum Evidence Unit (MEU). No finding without proof — enforced at runtime.

Multi-Provider LLM

Plug in OpenAI, Anthropic Claude, local Ollama, or use Mock mode for testing — all through a unified adapter.

FastAPI Server

HTTP API for team collaboration. POST /analyze with repo_url and receive structured research reports.

RAG Memory Store

Historical reports indexed via SQLite FTS5 full-text search. Past findings inform future analyses automatically.

Odin follows a layered architecture where each phase builds on the previous one. From low-level tool execution up through high-level research workflows.

Quick Example

Analyze a GitHub repository with a single command:

terminal 
# Set your API key
export OPENAI_API_KEY=sk-...

# Analyze a GitHub repository
python odin.py analyze https://github.com/owner/repo \
  --workflow vulnerability_research --provider openai

# Or analyze a local codebase (mock mode — no API key needed)
python odin.py analyze ./my-project --workflow codebase_research --provider mock
ℹ

No API key required for local analysis. Use --provider mock to run with simulated LLM responses. This is perfect for exploring Odin's capabilities without incurring API costs.

Output is a structured Markdown report with evidence-linked findings:

Evidence-Based Reasoning

Odin enforces a strict evidence policy: every conclusion must cite at least one Minimum Evidence Unit (MEU). This means every finding is traceable back to specific lines of code.

Code Snippet

Direct excerpts from source files with precise line numbers

Call Relations

Function call graphs with guard detection and taint tracking

Confidence Levels

High (≥0.8), Medium (0.5–0.8), Low (0.4–0.5), Uncertain (<0.4)
