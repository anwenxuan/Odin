Odin

Evidence-backed AI code analysis for systematic vulnerability research. Every finding, traceable to source.

Get Started View on GitHub

Stop trusting AI guesses.
Every finding backed by code.

Capabilities

Built for rigorous code analysis

Odin combines an autonomous agent loop with evidence enforcement, so every vulnerability claim is anchored to a code snippet you can verify.

MEU Enforcement

Every conclusion requires a Minimum Evidence Unit — file path, line range, and snippet. No evidence means no conclusion. No hallucinations.

DAG-Layer Parallelism

Independent workflow steps execute in parallel via ThreadPoolExecutor. An 8-step pipeline can complete in 3 layers instead of 8 sequential steps.

12 Built-in Skills

From repo mapping to exploit generation, Odin's 12 skills cover the full vulnerability research pipeline — each composable in any workflow.

MCP Integration

Connect Odin to GitHub, CVE databases, Jira, or any MCP-compatible data source. Structured context enrichment in real time.

RAG Knowledge Store

SQLite FTS5 full-text search over historical analysis reports. Ask questions against all past findings in seconds.

Evaluation Framework

Precision, Recall, F1 benchmarks against ground truth datasets. Track hallucination rates, tool call accuracy, and evidence coverage per release.

Quick Start

Up and running in 3 steps

No API key needed to try it out — mock mode works out of the box.

bash 
# 1. Install
pip install -e .
# or with uv
uv pip install -e .

# 2. Try without an API key
$ odin analyze ./examples --provider mock --verbose

  ✓ repo_discovery   [  1.2s ]
  ✓ entrypoints      [  0.8s ]
  ✓ call_graph       [  2.1s ]
  ✓ report           [  0.3s ]

  Found 3 entry points, 12 call chains, 2 auth guards.
  Report: .odin/audit/report-2026-03-26.md

# 3. With real AI (requires OPENAI_API_KEY)
export OPENAI_API_KEY=sk-...
$ odin analyze https://github.com/owner/repo \
    --workflow vulnerability_research \
    --provider openai --model gpt-4o-mini
