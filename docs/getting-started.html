Quick Start Guide

Getting Started

Python 3.11+ ~2 min to first analysis No API key needed for local mode

Prerequisites

Before installing Odin, make sure your system has:

Python 3.11+

Odin requires Python 3.11 or higher. Check your version: python3 --version

Git

Required for cloning remote repositories. Most systems have it pre-installed.

Optional: API Key

OpenAI or Anthropic API key for full AI analysis. Without it, use --provider mock.

Installation

1. Clone the Repository

terminal 
git clone https://github.com/your-org/odin.git
cd odin

2. Install Dependencies

terminal 
# Install in editable mode (recommended)
pip install -e .

# Or install all dependencies at once
pip install openai anthropic fastapi uvicorn pydantic pyyaml jsonschema

3. Verify Installation

terminal 
python odin.py --version
# Output: Odin v0.2.0
✓

Odin is installed and ready. You can now analyze codebases with a single command.

Quick Start

Option A: Analyze a Local Project (No API Key Needed)

Use mock mode to run with simulated AI responses — perfect for testing:

terminal 
python odin.py analyze ./my-project \
  --workflow codebase_research \
  --provider mock \
  --verbose

Option B: Analyze a GitHub Repository

Set your API key and analyze any public GitHub repo:

terminal 
# Set API key
export OPENAI_API_KEY=sk-...

# Analyze a GitHub repo
python odin.py analyze https://github.com/owner/repo \
  --workflow vulnerability_research \
  --provider openai \
  --model gpt-4o-mini

Option C: Analyze a Local Project with Real AI

Analyze a local codebase with full AI capabilities:

terminal 
export OPENAI_API_KEY=sk-...

python odin.py analyze ./my-project \
  --workflow vulnerability_research \
  --provider openai \
  --verbose

Understanding the Output

After analysis completes, Odin produces a structured Markdown report. Here's what each section means:

Section Description
Steps Each workflow step executed — shows status, tool calls, and iterations
entrypoints HTTP handlers, CLI commands, message listeners detected in the codebase
sinks Dangerous function calls identified (SQL execution, command run, file I/O)
hypotheses CWE-tagged vulnerability hypotheses generated from evidence
Evidence Total MEUs (Minimum Evidence Units) stored with line-level citations

Saving Reports

Output the report to a file instead of stdout:

terminal 
# Save as Markdown
python odin.py analyze ./my-project \
  --workflow vulnerability_research \
  --output ./report.md

# Save as JSON
python odin.py analyze ./my-project \
  --workflow vulnerability_research \
  --output ./report.json \
  --output-format json

Focusing on Specific Paths

When analyzing large repositories, narrow the scope to specific directories:

terminal 
python odin.py analyze https://github.com/owner/repo \
  --workflow vulnerability_research \
  --provider openai \
  --focus-paths src/auth src/api src/payment
ℹ

Focus paths are relative to the repository root. This significantly speeds up analysis for large codebases.

Choosing a Workflow

Odin ships with three built-in workflows for different analysis goals:

Workflow When to Use Time
codebase_research Quick overview of a codebase structure, entry points, and key components ~1-2 min
architecture_analysis Deep dive into architecture patterns, data flows, and module relationships ~3-5 min
vulnerability_research Full security audit: entry points → sinks → attack surfaces → PoC ~5-10 min

LLM Provider Configuration

Configure which AI model to use via environment variables or CLI flags:

.env 
# .env file (copy from .env.example)
OPENAI_API_KEY=sk-...
# ANTHROPIC_API_KEY=sk-ant-...

ODIN_PROVIDER=openai
ODIN_MODEL=gpt-4o-mini
ℹ

gpt-4o-mini is the recommended default — it's fast, affordable, and excellent for code analysis tasks.

Listing Available Skills & Workflows

terminal 
# List all available skills
python odin.py list-skills

# List all available workflows
python odin.py list-workflows
