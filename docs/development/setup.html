Development / Dev Setup

Local Development

Clone, set up, and run the full Odin stack from your terminal.

Prerequisites

Python

3.11 or later

Git

Latest version

pip / uv

Or any PEP 517 installer

Clone & Install

bash 
# 1. Clone the repository
git clone git@github.com:anwenxuan/Odin.git
cd odin

# 2. Create a virtual environment (macOS / Linux)
python -m venv venv
source venv/bin/activate

# On Windows:
# python -m venv venv
# venv\Scripts\activate

# 3. Install Odin in editable mode with dev dependencies
pip install -e ".[dev]"

# 4. Set up pre-commit hooks
pre-commit install
uv support — If you prefer uv, run uv sync --extra dev instead of pip install -e ".[dev]".

Running Tests

Odin uses pytest with the following conventions:

  • Unit tests live alongside source in tests/unit/
  • Integration tests live in tests/integration/
  • Fixtures are shared via tests/conftest.py
bash 
# Run all tests with verbose output
pytest tests/ -v

# Run only unit tests
pytest tests/unit/ -v

# Run with coverage report
pytest tests/ -v --cov=odin --cov-report=html

# Run tests matching a keyword
pytest tests/ -k sql_injection -v

# Run tests and stop on first failure
pytest tests/ -v -x

Linting & Type Checking

Odin enforces code quality with two tools:

  • ruff — fast linter and formatter (replaces flake8, isort, and black)
  • mypy — static type checker
bash 
# Run ruff linter (checks style, import order, unused vars, etc.)
ruff check .

# Format code with ruff (autofix safe issues)
ruff format .

# Run mypy type checker
mypy .

# Run both (CI equivalent)
ruff check . && mypy .

Pre-commit Hooks

After pre-commit install, these hooks run automatically before each git commit:

yaml 
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: check-yaml
      - id: end-of-file-fixer

  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.2.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

Local FastAPI Server

Odin ships with a FastAPI server for programmatic access and testing the REST API locally.

bash 
# Start the server with hot reload (for development)
uvicorn odin.server:app --reload --port 8000

# Server runs at http://localhost:8000
# API docs at http://localhost:8000/docs
# ReDoc at http://localhost:8000/redoc
API docs — FastAPI automatically generates interactive docs at /docs (Swagger UI) and /redoc. Use them to explore and test endpoints locally.

IDE Configuration

VSCode (Recommended)

Place this in .vscode/settings.json at the repository root:

json 
{
  "[python]": {
    "editor.defaultFormatter": "charliermarsh.ruff",
    "editor.formatOnSave": true,
    "editor.codeActionsOnSave": {
      "source.fixAll.ruff": "explicit"
    },
    "editor.rulers": [88, 120],
    "editor.defaultFoldingRangeProvider": "ms-python.vscode-pylance"
  },
  "python.analysis.typeCheckingMode": "basic",
  "python.analysis.extraPaths": ["."],
  "python.testing.pytestEnabled": true,
  "python.testing.unittestEnabled": false,
  "files.trimTrailingWhitespace": true,
  "files.insertFinalNewline": true
}

PyCharm

  • Set Python interpreter to the virtualenv's python (venv/bin/python)
  • Enable Ruff plugin and configure it as the default formatter
  • Set line length to 88 (matching ruff's default)
  • Enable pytest as the test runner in Settings → Tools → Python Integrated Tools

Environment Variables

Copy the example env file and fill in your credentials:

bash 
cp .env.example .env
bash 
# .env — never commit this file
ODIN_PROVIDER=openai
ODIN_MODEL=gpt-4o-mini
OPENAI_API_KEY=sk-your-key-here
ANTHROPIC_API_KEY=sk-ant-your-key-here   # optional
ODIN_WORKFLOW=vulnerability_research
ODIN_MOCK=false                          # true = skip AI, use mock mode
Never commit .env — It is listed in .gitignore. API keys and secrets must never be pushed to version control.
