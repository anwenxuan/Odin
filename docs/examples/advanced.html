Advanced Examples

Custom workflows, MCP tool integration, batch analysis, and CI/CD automation for enterprise security teams.

Scenario

Your enterprise runs a multi-repo monorepo containing authentication services, payment APIs, and data pipelines. You need a systematic, repeatable security audit process that runs on every pull request and produces a consolidated report for the security team.

Custom Workflow YAML

Define a tailored enterprise_security.yaml workflow that combines multiple skills, configures DAG-layer parallelism, and integrates MCP tools:

yaml 
# .odin/workflows/enterprise_security.yaml
# Place in your project root or ~/.odin/workflows/

name: enterprise_security
version: "1.0"
description: "Enterprise security audit: full-stack analysis with MCP tools"

# Global settings
settings:
  provider: openai
  model: gpt-4o
  parallelism: 4                  # Max parallel branches in DAG
  meu_strict: true                # Reject findings without full MEU evidence
  confidence_threshold: 0.6         # Only report findings with confidence ≥ 60%
  timeout_per_skill: 120             # Seconds per skill before auto-cancel

# MCP external tools (see MCP Integration section below)
mcp:
  enabled: true
  servers:
    - name: gh_codeql
      command: npx @github/codeql-runner
      args: [--db=$DB_PATH]
    - name: semgrep
      command: semgrep --config=auto --json

# DAG Layer 1: Discovery (sequential)
layers:
  - name: layer_1_discovery
    mode: sequential
    skills:
      - skill: repo_discovery
        params:
          depth: full
          languages: [python, typescript, go, java]

  # DAG Layer 2: Surface analysis (parallel branches)
  - name: layer_2_surface
    mode: parallel
    skills:
      - skill: entrypoints
        params: {}
      - skill: call_graph
        params:
          max_depth: 10
          include_stdlib: false
      - skill: dependency_scout
        params:
          check_known: true
          advisory_db: osv

  # DAG Layer 3: Vulnerability scanning (parallel)
  - name: layer_3_vulns
    mode: parallel
    skills:
      - skill: sql_injection
        params:
          sensitivity: high
          context_lines: 5
      - skill: xss
        params:
          contexts: [html, js, attr, template]
      - skill: command_injection
        params:
          shells: [bash, sh, zsh, powershell]
      - skill: auth_guard
        params:
          check_all_endpoints: true
      - skill: secret_scanner
        params:
          patterns: [aws_key, gcp_token, slack_webhook, jwt_hs256]
          exclude_paths: [*.test.*, *_test.go, node_modules/]

  # DAG Layer 4: Reporting (sequential)
  - name: layer_4_report
    mode: sequential
    skills:
      - skill: report_generator
        params:
          format: [markdown, json]
          output_dir: .odin/audit
          include_call_chains: true
          include_meu_bundle: true
      - skill: slack_notifier
        params:
          webhook_url: $SLACK_WEBHOOK_URL
          channel: "#security-audit"
          only_high: false

MCP Tool Integration

The Model Context Protocol (MCP) allows Odin to delegate analysis to external security tools — CodeQL, Semgrep, Trivy — and ingest their results as evidence-backed MEUs.

MCP Configuration

yaml 
# .odin/mcp_config.yaml
version: "1.0"

servers:
  # GitHub CodeQL — used for deep semantic analysis on Python/JS/Go
  codeql:
    type: process
    command: codeql
    args:
      - database
      - create
      - --language=python
      - --source-root=$REPO_PATH
      - $DB_PATH
    meu_mapping:              # Map CodeQL query results → MEU schema
      severity: query_severity
      cwe: query.cwe.id
      evidence: results[].code_snippet

  # Semgrep — fast rule-based scanning
  semgrep:
    type: process
    command: semgrep
    args:
      - --config
      - p/security-audit
      - --json
      - --no-git-ignore
      - $REPO_PATH
    meu_mapping:
      severity: extra.severity
      cwe: extra.metadata.cwe[]
      evidence: extra.lines

  # Trivy — dependency vulnerability scanning
  trivy:
    type: process
    command: trivy
    args:
      - fs
      - --format
      - json
      - --ignore-unfixed
      - $REPO_PATH
    meu_mapping:
      severity: Results[].Vulnerabilities[].Severity
      cwe: Results[].Vulnerabilities[].CWEIDs[]
      evidence: Results[].Vulnerabilities[].FixedVersion

Batch Analysis & Aggregated Report

For monorepos or fleet-wide audits, use a shell script to iterate over multiple repositories and produce a consolidated JSON report:

bash 
#!/usr/bin/env bash
# scripts/batch_audit.sh — Run security audits across multiple repos

set -euo pipefail

REPOS=("auth-service" "payment-api" "data-pipeline" "user-service")
OUTPUT_DIR=.odin/batch-reports
TIMESTAMP=$(date +%Y-%m-%d_%H-%M)

mkdir -p "$OUTPUT_DIR/$TIMESTAMP"

for repo in "${REPOS[@]}"; do
  echo "━━━ Auditing $repo ━━━"

  odin analyze "https://github.com/acme/$repo.git" \
    --workflow enterprise_security \
    --provider openai \
    --output "$OUTPUT_DIR/$TIMESTAMP/$repo" \
    --format json \
    2>&1 | tee "$OUTPUT_DIR/$TIMESTAMP/$repo.log"

done

# Aggregate all findings into a unified report
odin aggregate \
  --input-dir "$OUTPUT_DIR/$TIMESTAMP" \
  --output "$OUTPUT_DIR/$TIMESTAMP/_summary.json" \
  --format json

echo "Batch audit complete. Reports: $OUTPUT_DIR/$TIMESTAMP/"

CI/CD Integration

Run Odin as a required check in your pull request pipeline. Fail on high/critical findings:

yaml 
# .github/workflows/security-audit.yml
name: Security Audit

on:
  pull_request:
    branches: [main, develop]
  push:
    branches: [main]
  schedule:
    # Weekly full monorepo audit
    - cron: "0 3 * * 1"

env:
  ODIN_PROVIDER: openai
  ODIN_MODEL: gpt-4o-mini

jobs:
  odin-audit:
    name: Odin Security Audit
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0   # Shallow clones don't show changed files

      - name: Detect changed files
        id: changed
        uses: tj-actions/changed-files@v44
        with:
          files_from_commits: "${{ github.event.pull_request.base.sha }}"

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install Odin
        run: |
          pip install -e ".[dev]"

      - name: Run Odin audit
        id: odin
        run: |
          if [[ "${{ github.event_name }}" == "pull_request" ]]; then
            # PR mode: diff only the changed files for speed
            odin analyze ./ \
              --workflow enterprise_security \
              --diff-since "${{ github.event.pull_request.base.sha }}" \
              --provider openai \
              --output .odin/audit \
              --fail-on high \
              --slack-webhook "${{ secrets.SLACK_WEBHOOK_URL }}"
          else
            # Scheduled or push: full scan
            odin analyze ./ \
              --workflow enterprise_security \
              --provider openai \
              --output .odin/audit \
              --fail-on critical
          fi

      - name: Upload audit report
        uses: actions/upload-artifact@v4
        with:
          name: odin-audit-report
          path: .odin/audit/

      - name: Post comment on PR
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## 🔍 Odin Security Audit\n\nAudit complete. See the [full report](https://github.com/${{ context.repo.owner }}/${{ context.repo.repo }}/actions/runs/${{ github.run_id }}) for details.\n\nSummary: **${{ steps.odin.outputs.summary }}**`
            })
PR diff mode — Use --diff-since to limit analysis to changed files since the merge base. This dramatically reduces cost and analysis time for large monorepos, typically completing in under 60 seconds.
