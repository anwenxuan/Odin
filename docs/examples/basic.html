Examples / Basic Usage

Basic Example

A complete walkthrough — from installation to generated security report — in under 10 minutes.

Scenario

You maintain a small Python web application and want to audit it for SQL injection vulnerabilities before shipping a new release.

This example uses mock mode so you can run it without an API key. Replace --provider mock with --provider openai (and set OPENAI_API_KEY) for real AI-powered analysis.

The target repository we'll use is a simplified Flask app with intentionally planted SQL injection patterns:

bash 
git clone https://github.com/anwenxuan/odin-test-repo.git
cd odin-test-repo

The Command

Run the vulnerability_research workflow against the repository:

bash 
$ odin analyze ./ \
    --workflow vulnerability_research \
    --provider mock \
    --skills sql_injection auth_guard \
    --output .odin/audit \
    --verbose

MEU Output Format

Each finding is stored as a Minimum Evidence Unit (MEU) in .odin/audit/evidence.jsonl. Every MEU is self-contained and traceable.

json 
{
  "meu_id": "MEU-2026-0326-0042",
  "skill": "sql_injection",
  "timestamp": "2026-03-26T14:32:07Z",
  "severity": "high",
  "confidence": 0.87,
  "location": {
    "file": "app.py",
    "line": 42,
    "function": "get_user",
    "endpoint": "GET /api/users"
  },
  "evidence_snippet": "cursor.execute(f'SELECT * FROM users WHERE id = {user_id}')",
  "call_chain": [
    "get_user (app.py:42)",
    "db_query (db.py:15)",
    "cursor.execute (psycopg2:intern)"
  ],
  "taint_path": {
    "source": "request.args['id']",
    "sanitizer": null,
    "sink": "cursor.execute"
  },
  "recommendation": "Use parameterized queries: cursor.execute('SELECT * FROM users WHERE id = %s', (user_id,))",
  "cwe": "CWE-89"
}

MEU Field Reference

Field Description
meu_id Globally unique identifier: MEU-YYYY-MMDD-NNNN
skill Which Skill produced this finding
confidence 0.0 – 1.0 score; reflects certainty of the finding
severity One of: critical, high, medium, low, info
call_chain Full path from entry point to vulnerable sink
taint_path Source → (no sanitizer) → sink triple
evidence_snippet Exact code snippet cited as evidence (quoted verbatim)
cwe CWE identifier (if applicable)

Generated Report

The markdown report at .odin/audit/report-*.md follows this structure:

markdown 
# Security Audit Report
Repository: odin-test-repo
Date: 2026-03-26
Workflow: vulnerability_research
Provider: mock

---

## Summary

| Metric | Value |
|--------|-------|
| Entry points | 4 |
| Call chains | 22 |
| Auth guards found | 3 |
| Auth guards missing | 1 |
| **SQL injection findings** | **2 (HIGH)** |
| Total MEUs | 9 |

---

## Finding #1 — SQL Injection (HIGH)

**Location:** `app.py:42` — `get_user()` — `GET /api/users`

**Evidence:**
```python
cursor.execute(f'SELECT * FROM users WHERE id = {user_id}')
```

**Taint path:** `request.args['id']` → *(no sanitizer)* → `cursor.execute`

**Call chain:**
```
get_user (app.py:42)
  └→ db_query (db.py:15)
      └→ cursor.execute (psycopg2:intern)
```

**Recommendation:** Use parameterized queries:
```python
cursor.execute('SELECT * FROM users WHERE id = %s', (user_id,))
```

**Confidence:** 87% | **CWE:** CWE-89 | **MEU:** MEU-2026-0326-0042

---

## Finding #2 — Missing Authentication Guard (MEDIUM)

**Location:** `app.py:78` — `GET /api/search`

**Finding:** This endpoint has no authentication check.

**Recommendation:** Add `@require_auth` decorator or inline auth guard.

**MEU:** MEU-2026-0326-0043

---

## Evidence Store

Full evidence units: `.odin/audit/evidence.jsonl`

Interpreting Results

HIGH / CRITICAL

Act immediately. These findings indicate exploitable vulnerabilities with high confidence. Do not ship until resolved.

MEDIUM

Address before production release. Confirm via manual testing; some may be false positives in edge cases.

LOW / INFO

Best-practice improvements. Not urgent but recommended for defense-in-depth.

Confidence score — Not all findings are equal. A confidence: 0.87 finding has been validated against multiple heuristics. Low-confidence findings (< 0.5) should be manually reviewed before acting on them.
