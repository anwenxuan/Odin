CLI Overview

The Odin CLI provides a unified interface for analyzing codebases. It is invoked via the odin.py entry point:

odin.py
# As a module
python odin.py <command> [options]

# After pip install (if configured)
odin <command> [options]
ℹ

All commands support --version to display the installed Odin version (v0.2.0).

Global Options

FlagDescription
--versionShow version number and exit
-v, --verboseEnable verbose (DEBUG level) logging

odin analyze

The primary command for analyzing repositories. Supports both local paths and GitHub URLs.

cli/main.py
odin analyze <repo> [options]

Positional Arguments

ArgumentTypeDescription
repostringGitHub URL or local path to the repository

Options

FlagDefaultDescription
--workflowcodebase_researchAnalysis workflow: vulnerability_research, codebase_research, architecture_analysis
--provideropenaiLLM provider: openai, anthropic, ollama, mock
--modelgpt-4o-miniModel name (e.g. gpt-4o-mini, claude-sonnet-4-20250514)
--output(stdout)Output file path for the report (Markdown or JSON)
--output-formatmarkdownReport format: markdown or json
--focus-paths(none)Space-separated list of paths to focus analysis on (relative to repo root)
--max-iterations20Maximum tool call iterations per Skill step
--branch(default)Git branch to clone (only for GitHub URLs)
--no-cachefalseForce re-clone of remote repositories (ignore cache)
--verbosefalsePrint detailed execution logs
--servefalseStart FastAPI server mode instead of running analysis
--serve-port8080Port for API server mode

Examples

Basic usage with mock mode (no API key needed):

terminal
python odin.py analyze ./my-project --provider mock

GitHub repository with OpenAI:

terminal
python odin.py analyze https://github.com/owner/repo \
  --workflow vulnerability_research \
  --provider openai \
  --model gpt-4o-mini

Local repository with focus paths:

terminal
python odin.py analyze ./my-project \
  --workflow architecture_analysis \
  --focus-paths src/auth src/api \
  --verbose

Save report to file:

terminal
# Save as Markdown
python odin.py analyze ./repo \
  --workflow vulnerability_research \
  --output ./report.md

# Save as JSON
python odin.py analyze ./repo \
  --workflow vulnerability_research \
  --output ./report.json \
  --output-format json

With specific branch:

terminal
python odin.py analyze https://github.com/owner/repo \
  --branch develop \
  --workflow codebase_research \
  --provider anthropic

odin list-workflows

Lists all registered Workflows and their step chains.

cli/commands/list_workflows.py
odin list-workflows

Example output:

terminal

  repo_discovery → entrypoints_detection → call_graph_trace → data_structure_extraction → report_generation

Workflow: architecture_analysis (7 steps)
  repo_discovery → entrypoints_detection → call_graph_trace → data_structure_extraction → auth_logic_detection → dependency_analysis → report_generation

  [call_graph_trace, auth_logic_detection] → input_flow_analysis →
  attack_surface_mapping → vulnerability_hypothesis → exploit_generation → report_generation

Exit Codes

CodeMeaning
0Success — analysis completed without errors
1Error — repository not found, workflow not found, or internal error
130Interrupted — user pressed Ctrl+C

Complete Workflow Examples

Security Audit Workflow

terminal
# Full security audit with OpenAI GPT-4o mini
export OPENAI_API_KEY=sk-...

# Analyze a production codebase
python odin.py analyze https://github.com/myorg/api-service \
  --workflow vulnerability_research \
  --provider openai \
  --output ./security-audit-$(date +%Y%m%d).md \
  --verbose

PR Review Workflow (CI/CD Integration)

terminal
# Fast PR review using codebase_research
python odin.py analyze ./changes \
  --workflow codebase_research \
  --provider openai \
  --output ./pr-review.json \
  --output-format json \
  && echo "Analysis complete" || echo "Analysis failed with code $?"

Local Development (No API Key)

terminal
# Test locally without API costs
python odin.py analyze ./my-local-repo \
  --workflow codebase_research \
  --provider mock \
  --verbose
