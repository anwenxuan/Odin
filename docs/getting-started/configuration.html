Get Started / Configuration

Configuration

Environment variables, odin.yaml, provider settings, and output directory.

Environment Variables

Odin is configured primarily through environment variables. You can set these in your shell, a .env file, or your CI/CD pipeline.

Variable Description Default
OPENAI_API_KEY OpenAI API key for GPT-4o, GPT-4o-mini, and other OpenAI models. Get one at platform.openai.com. None
ANTHROPIC_API_KEY Anthropic API key for Claude 3.5 Sonnet, Claude 3 Opus, and other Anthropic models. Get one at console.anthropic.com. None
ODIN_PROVIDER The default AI provider to use. Valid values: openai, anthropic, ollama, mock. mock
ODIN_MODEL Default model when no model is specified on the command line. Example: gpt-4o-mini or claude-3-5-sonnet-20241022. Provider default
ODIN_MAX_TOKENS Maximum number of tokens to generate in a single LLM response. Reduce to control costs; increase for detailed analysis. 4096
ODIN_OUTPUT_DIR Directory where analysis reports and evidence stores are written. Can be an absolute or relative path. .odin/audit
ODIN_TEMPERATURE LLM sampling temperature (0.0 to 1.0). Lower values make output more deterministic; higher values add creativity. 0.1
ODIN_VERBOSE Enable verbose logging. Set to 1 or true for detailed debug output. 0
ODIN_RAG_ENABLED Enable RAG knowledge store for querying past analysis results. Set to 0 to disable. 1
Priority order
Configuration is resolved in this order (highest to lowest): CLI flags > odin.yaml > environment variables > defaults.

Configuration File (odin.yaml)

Place an odin.yaml file in your project root or home directory for persistent configuration.

yaml 
# odin.yaml
# Place in your project root or ~/.odin/odin.yaml

# Default provider (openai | anthropic | ollama | mock)
provider: openai

# Default model
model: gpt-4o-mini

# Max tokens per LLM response
max_tokens: 4096

# Temperature (0.0 - 1.0)
temperature: 0.1

# Output directory for reports and evidence
output_dir: .odin/audit

# Enable RAG knowledge store
rag:
  enabled: true
  db_path: ~/.odin/knowledge.db

# MCP server integrations
mcp:
  servers:
    - name: github
      command: npx @modelcontextprotocol/server-github
      env:
        GITHUB_PERSONAL_ACCESS_TOKEN: ${GITHUB_TOKEN}
    - name: filesystem
      command: npx @modelcontextprotocol/server-filesystem
      args:
        - /tmp/odin-sandbox

# Workflow defaults
workflow:
  default: vulnerability_research
  timeout: 600
  retry:
    max_attempts: 3
    backoff: exponential

# Sandbox isolation (Docker)
sandbox:
  enabled: false
  image: odin-sandbox:latest

Provider Configuration

Odin supports multiple AI providers. Choose based on your needs and budget.

OpenAI

Recommended

Best overall performance and cost efficiency. Supports gpt-4o, gpt-4o-mini, gpt-4-turbo.

bash 
export OPENAI_API_KEY=sk-your-key-here
$ odin analyze ./target --provider openai --model gpt-4o-mini

Anthropic

Extended Context

Excellent for complex reasoning and code understanding. Supports claude-3-5-sonnet-20241022, claude-3-opus-20240229. 200K context window on Sonnet.

bash 
export ANTHROPIC_API_KEY=sk-ant-your-key-here
$ odin analyze ./target --provider anthropic --model claude-3-5-sonnet-20241022

Ollama

Local / Offline

Run open-source models locally (Llama 3, CodeLlama, Qwen2, etc.). Zero API costs, full privacy. Requires Ollama installed and running.

bash 
# Pull a model first
ollama pull llama3.1

$ odin analyze ./target --provider ollama --model llama3.1

Mock

No API Key

Rule-based simulation of the full pipeline. No API key required. Perfect for learning, CI/CD testing, and workflow development.

bash 
$ odin analyze ./target --provider mock --verbose

Output Directory

By default, Odin writes all outputs to .odin/audit/ relative to where you run the command.

Reports

.odin/audit/report-YYYY-MM-DD.md

Human-readable Markdown report with all findings, evidence snippets, and confidence scores.

Evidence Store

.odin/audit/evidence.jsonl

JSON Lines file containing every MEU (Minimum Evidence Unit) collected during the run.

Session Log

.odin/audit/session-YYYY-MM-DD.jsonl

Complete execution log including layer timings, tool calls, LLM prompts, and responses.

Override the output directory with the ODIN_OUTPUT_DIR environment variable or the --output CLI flag:

bash 
# Via environment variable
export ODIN_OUTPUT_DIR=/var/results/odin

# Via CLI flag (overrides env)
$ odin analyze ./target --output /tmp/my-results
