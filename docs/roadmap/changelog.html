Release History

Changelog

All notable changes to the Odin project, following the Keep a Changelog standard.

This project adheres to Semantic Versioning. Breaking changes will be clearly marked. The current stable release is v0.2.0.

v0.2.0

(June 2024) Current Release
Added

  • Agent loop with checkpoint support — ReAct-style reasoning with configurable thought/action/observation cycles and state snapshots at each step.

  • DAG parallel execution — Layer-based task graph that schedules independent tasks simultaneously across DAG layers, dramatically reducing total analysis time.

  • MCP integration — Model Context Protocol support for dynamic tool discovery, external resource connections, and extensible agent capabilities.

  • FastAPI REST server — Production-ready API with workflow management, session persistence, streaming responses, and OpenAPI documentation.

  • RAG knowledge store — Vector-backed retrieval for persisting vulnerability findings, CVE records, and cross-session research context with semantic search.

  • Evaluation framework with benchmark suite — Quantitative measurement of agent accuracy, recall, and MEU compliance using labeled CVE datasets.

Changed

  • Refactored the ToolExecutor interface to support async/await patterns throughout the execution pipeline.

  • Updated YAML workflow schema to support DAG definitions, layer assignments, and parallel execution hints.

  • Upgraded FastAPI server with Pydantic v2 models, streaming responses, and structured logging.

Fixed

  • Corrected tool timeout handling that caused silent failures in long-running skill executions.

  • Fixed state serialization issue when resuming agent sessions from checkpoint snapshots.

  • Resolved race condition in parallel skill execution that could produce duplicate findings.

v0.1.0

(January 2024)
Added

  • Tool executor with 12 built-in skills — Grep, AST, Semgrep, SBOM, CVSS, CWE, Deps, License, Secrets, Malware, Fuzz, and CVE Lookup.

  • 3 analysis workflows — Scan, Deep Audit, and CVE Triage — each orchestrating a tailored sequence of skills.

  • CLI interface — Command-line tool with workflow selection, target specification, verbose output, and JSON result export.

  • YAML workflow definitions — Human-readable configuration files defining skill sequences, parameters, and pass-through logic.

  • Mock provider for offline testing — Local mock LLM provider requiring no API keys, enabling full offline development and testing.

  • MEU (Minimum Evidence Unit) enforcement — Every finding must include structured evidence: location, severity, and reproduction steps.

Changed

  • Initial public release — established core architecture, skill interface, and workflow engine.
