Vision & Strategy

Where Odin is Heading

An evidence-backed autonomous vulnerability research framework — from today’s foundation to tomorrow’s production-grade CVE pipelines.

In 3–5 years, Odin powers automated vulnerability research pipelines at scale — from initial repo scan to CVE submission, fully autonomous with human oversight.

Clarity

What Odin Will Not Do

Clear boundaries prevent scope creep and keep the project focused.

Odin has a focused, well-defined scope. The following are explicitly not on the roadmap and will not be added:

Web UI

Odin is a CLI-first and API-first tool. There are no plans to build a web-based dashboard or UI.

SaaS Hosting

Odin will remain a self-hosted, open-source project. No managed cloud service or SaaS offering is planned.

Non-Code Analysis

While Odin uses SBOMs and binary metadata, it focuses exclusively on software vulnerability research — not general document or image analysis.

Proprietary Integrations

Odin will not add integrations that require proprietary licenses or vendor lock-in. All integrations are open and community-driven.
